# Receiving with Webhooks autoconfig

Webhooks autoconfig is a convenient way to configure your webhook endpoints in a code-first way, using the Svix SDK.

Key benefits:
- Define the webhook configuration in code (rather than both code and the webhooks management dashboard).
- Automatically update URL, event types and other endpoint settings.
- Signature verification is supported out of the box, no additional secrets to wire up.

## How to use Webhooks autoconfig

1. Create an endpoint and select autoconfig as the configuration method.
2. Get the autoconfig token.
3. Set the endpoint configuration and the autoconfig token in your application code or CI/CD pipeline.
4. Call `subscribe()` to complete the configuration.

### Payload Verification

When using Webhooks autoconfig, the payload verification is supported out of the box, no need to wire up additional secrets.

To verify the payload, call verify() on the same AutoConfig instance you used to subscribe. The autoconfig token contains the secret key to verify the payload.

```typescript
const webhook = new Svix.AutoConfig({
  token: process.env.AUTO_CONFIG_TOKEN,
  ...
});

webhook.verify(payload, headers);
```

For more details on how to verify the payload and headers, see [Verifying Webhooks](/receiving/verifying-payloads/how).

## Integration Examples

AutoConfig can be used in a variety of ways, as part of a CI/CD pipeline, a serverless function, or the startup process of your application.

Here are some examples in different frameworks and deployment platforms.

### In Github Actions (CI/CD)

```yaml
name: deploy
on:
  push:
    branches: [main]
jobs:
  subscribe-endpoint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
      - run: npm ci
      - run: |
          node --input-type=module <<'EOF'
          import { Svix } from "svix";

          const webhook = new Svix.AutoConfig({
            token: process.env.AUTO_CONFIG_TOKEN,
            eventTypes: ["invoice.paid", "user.created"],
            url: "https://api.us.example.com/webhooks/acme",
          });

          await webhook.subscribe();
          EOF
        env:
          AUTO_CONFIG_TOKEN: ${{ secrets.AUTO_CONFIG_TOKEN }}
```

### Go (Standard lib)

```go
package main

import (
	"context"
	"log"
	"net/http"
	"os"

	svix "github.com/svix/svix-webhooks/go"
)

func main() {
	ctx := context.Background()

	webhook, err := svix.NewAutoConfig(svix.AutoConfigOptions{
		Token:      os.Getenv("AUTO_CONFIG_TOKEN"),
		EventTypes: []string{"invoice.paid", "user.created"},
		URL:        "https://api.us.example.com/webhooks/acme",
	})
	if err != nil {
		log.Fatal(err)
	}
	if err := webhook.Subscribe(ctx); err != nil {
		log.Fatal(err)
	}

	http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		w.WriteHeader(http.StatusOK)
	})
	log.Fatal(http.ListenAndServe(":8080", nil))
}
```

### Rust (axum)

```rust
use std::env;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let webhook = svix::AutoConfig::new(svix::AutoConfigOptions {
        token: env::var("AUTO_CONFIG_TOKEN")?,
        event_types: vec![
            "invoice.paid".to_string(),
            "user.created".to_string(),
        ],
        url: "https://api.us.example.com/webhooks/acme".to_string(),
    });

    webhook.subscribe().await?;

    let app = axum::Router::new().route("/", axum::routing::get(|| async { "ok" }));

    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await?;
    axum::serve(listener, app).await?;

    Ok(())
}
```
