# Webhooks Auto-Config

Webhooks Auto-Config is a declarative, code-first way to configure webhook endpoints, using the Svix SDK.

Key benefits for your customers:
- Define the webhook configuration in code (rather than both code and the app portal UI).
- Automatically update URL, event types and other endpoint settings.
- Signature verification is supported out of the box, no additional secrets to wire up.

## How it works

When Auto-Config is enabled in your environment, your users will be able to configure their webhook endpoints in the Application Portal.
The portal will issue an **auto-config token**, which can be used to configure the endpoint in code.

Using the Svix SDK `WebhooksAutoConfig` client, the consumer specifies the endpoint configuration, passes the token and calls `subscribe()`.
When `subscribe()` is called, Svix updates the endpoint to match the configuration.

This can be done in a CI/CD pipeline, a serverless function, or the startup process of your application.

### Updating the endpoint configuration

To update the endpoint configuration, your customers simply need to change the WebhooksAutoConfig options and call `subscribe()` again.
No need for any manual changes in the Application Portal UI.

## Using Webhooks Auto-Config in CI/CD

The recommended way to use Auto-Config is to include it as part of the webhook consumer's deployment process, so the endpoint configuration is updated on every deploy.

For additional examples on how to set it up, see [Receiving with Webhooks Auto-Config](/receiving/webhooks-auto-config).


## Enabling Auto Configuration

Turn on endpoint auto configuration in your environment’s settings on the Svix dashboard.
After it is enabled, App Portal users can start using Auto Configuration, with no additional setup required.
