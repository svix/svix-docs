# Webhooks AutoConfig

Webhooks AutoConfig is a declarative, code-first way to configure webhook endpoints, using the Svix SDK. It makes it
significantly easier for your customers to configure webhook endpoints, and ensures that their configuration is always in sync
with the latest version of the application code, without additional manual steps.

Key benefits for your customers:
- Define the webhook configuration in code. No need for manual updates int the webhooks management UI.
- Automatically update URL, event types and other endpoint settings. No risk of configuration drift.
- Signature verification is supported out of the box. No additional secrets to wire up.

## How it works

When AutoConfig is enabled in your environment, your users will be able to choose it as an option when creating an endpoint in the Application Portal.

![AutoConfig option in the Application Portal](/img/autoconfig/autoconfig-create.png)

The portal will issue an *AutoConfig token*, which can be used to configure the endpoint with the AutoConfig SDK.

The AutoConfig token also contains the endpoint signing secret, so the same instance can be used to verify webhook signatures.

Here's an example of how this looks like in code:

```js
import { AutoConfig } from "svix";

const webhook = new AutoConfig({
  token: AUTO_CONFIG_TOKEN,
  eventTypes: ["invoice.paid", "user.created"],
  url: "https://api.us.example.com/webhooks/acme",
});

// Update the endpoint configuration
await webhook.subscribe();

// Same AutoConfig instance can be used to verify webhook signatures
const isVerified = webhook.verify(payload, headers);
```

To update the endpoint configuration, your customers simply need to change the AutoConfig options and call `subscribe()` again, which can be done as part of their
application startup or deployment process, so there is no need for manual steps in the Application Portal UI.

For more information on how to receive webhooks using AutoConfig, see [Receiving with Webhooks AutoConfig](/receiving/webhooks-autoconfig).
